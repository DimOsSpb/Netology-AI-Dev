## Для организации монорепозитория (нескольких проектов с общим окружением) в связке с редактором Zed связка инструментов uv + ruff является лучшим решением в 2026 году по скорости, простоте настройки и интеграции.
Ниже представлена оптимальная архитектура, методика работы и разбор возможностей инструментов.
------------------------------
## Архитектура монорепозитория
Рекомендуется использовать структуру uv Workspace (рабочее пространство). Она позволяет иметь общую виртуальную среду (.venv), общие зависимости, но разделять код на независимые проекты (пакеты).

my-monorepo/
├── .venv/                  # Единое виртуальное окружение для всего репозитория
├── .zed/                   # Настройки для редактора Zed
│   └── settings.json       # Конфигурация LSP, Ruff и путей к интерпретатору
├── apps/                   # Папка для конечных приложений / сервисов
│   ├── app_one/
│   │   ├── src/
│   │   └── pyproject.toml  # Специфичные зависимости app_one
│   └── app_two/
│       ├── src/
│       └── pyproject.toml
├── libs/                   # Папка для переиспользуемых модулей (shared library)
│   └── core_shared/
│       ├── src/
│       └── pyproject.toml
├── pyproject.toml          # Корневой файл: объявляет workspace и общие dev-зависимости
├── ruff.toml               # Глобальные правила линтинга и форматирования
└── uv.lock                 # Единый lock-файл для гарантированной идентичности версий

------------------------------
## Методика настройки (Шаг за шагом)## 1. Инициализация воркспейса
В корне репозитория создайте файл pyproject.toml, который объединит все проекты:

[project]
name = "my-monorepo"
version = "0.1.0"
requires-python = ">=3.11"
dependencies = [] # Сюда можно добавить общие зависимости (например, loguru)

[tool.uv.workspace]
members = ["apps/*", "libs/*"]

[tool.uv.dependencies]# Общие инструменты разработки для всех подпроектов
ruff = ">=0.3.0"
pytest = ">=8.0.0"

## 2. Связывание проектов внутри монорепозитория
Если app_one использует общую библиотеку core_shared, пропишите это в apps/app_one/pyproject.toml через локальный путь:

[project]
name = "app_one"
version = "0.1.0"
dependencies = [
    "core_shared", # Имя пакета из libs/core_shared
]

[tool.uv.sources]
core_shared = { workspace = true }

## 3. Синхронизация окружения
Запустите команду в корне:

uv sync

uv мгновенно создаст одну папку .venv в корне, установит туда зависимости всех приложений и свяжет core_shared в режиме editable (изменения в библиотеке сразу видны в приложении).
------------------------------
## Настройка IDE Zed
Zed поддерживает интеграцию с Pyright (LSP) и Ruff из коробки. Чтобы всё работало идеально в рамках монорепозитория, создайте файл .zed/settings.json:

{
  "languages": {
    "Python": {
      "language_servers": ["pyright", "ruff"],
      "format_on_save": "on",
      "formatter": {
        "external": {
          "command": "uv",
          "arguments": ["run", "ruff", "format", "--stdin-filename", "{buffer_path}"]
        }
      }
    }
  },
  "lsp": {
    "pyright": {
      "settings": {
        "python": {
          "pythonPath": ".venv/bin/python" 
        }
      }
    },
    "ruff": {
      "binary": {
        "path": "uv",
        "arguments": ["run", "ruff", "server"]
      }
    }
  }
}

Примечание для Windows: замените путь к интерпретатору на ".venv/Scripts/python.exe".

## Что дают uv и ruff?
Использование этих инструментов от компании Astral качественно меняет DX (Developer Experience):
## Что дает uv?

   1. Экстремальная скорость: Написан на Rust. Установка зависимостей, создание .venv и сборка происходят в 10–100 раз быстрее, чем у pip, poetry или pipenv.
   2. Управление версиями Python: Команда uv python pin 3.12 сама скачает нужную версию Python, вам больше не нужен pyenv.
   3. Единый Lock-файл (uv.lock): Гарантирует, что у всех разработчиков и на CI/CD развернуты абсолютно одинаковые версии библиотек для всех подпроектов.
   4. Умный кэш: Если 5 ваших подпроектов используют pydantic, uv скачает его один раз и сделает hard-link. Место на диске экономится колоссально.

## Что дает ruff?

   1. Замена 10+ инструментов: Один ruff заменяет flake8, black, isort, autoflake, mccabe, pep8-naming и плагины безопасности (типа bandit).
   2. Мгновенный линтинг: Работает в сотни раз быстрее аналогов (тоже написан на Rust). Ошибки и неправильные импорты подсвечиваются в Zed на лету, без задержек.
   3. Авто-фикс (Auto-fixing): Не просто подсвечивает проблемы, а сам исправляет неиспользуемые импорты, неверный порядок импортов, устаревший синтаксис (например, переводит List[str] в list[str]) при сохранении файла в Zed.
